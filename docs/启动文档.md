# 知枢星图启动指南

## 环境要求

### 后端环境

- Python 3.13 (Anaconda)
- Windows 系统
- 已安装 Anaconda (F:\ANACONDA)

### 前端环境

- Node.js 18+ (推荐使用 LTS 版本)
- npm 或 pnpm 包管理器

***

## 快速启动

### 1. 首次运行：安装依赖

#### 后端依赖

**重要**：必须以管理员身份运行 Anaconda Prompt！

1. 在 Windows 开始菜单搜索 "Anaconda Prompt"
2. 右键选择 "以管理员身份运行"
3. 执行以下命令安装依赖：

```bash
pip install python-jose cryptography passlib python-multipart bcrypt uvicorn fastapi sqlalchemy pydantic pydantic-settings python-dotenv
```

#### 前端依赖

在项目根目录打开终端，进入 frontend 目录安装依赖：

```powershell
cd d:\知枢星图\frontend
npm install
```

### 2. 启动服务

#### 启动后端服务

**推荐方式**：在 PowerShell 终端中运行

```powershell
cd d:\知枢星图\backend
$env:PYTHONPATH = "D:\知枢星图\backend"
F:\ANACONDA\python.exe -m uvicorn app.main:app --reload --port 8000
```

或双击运行 `backend/start.bat` 文件。

服务启动成功后，访问 <http://localhost:8000/health> 查看 API 状态。

启动成功标志：

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started server process [xxxxx]
INFO:     Waiting for application startup.

```

#### 启动前端服务

在新的终端窗口中运行：

```powershell
cd d:\知枢星图\frontend
npm run dev
```

启动成功后访问 <http://localhost:5173>

启动成功标志：

```

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

### 3. 完整启动顺序

1. **先启动后端**：运行 `backend/start.bat`
2. **再启动前端**：在 frontend 目录运行 `npm run dev`
3. **访问应用**：打开浏览器访问 <http://localhost:5173>

### 4. 常见问题：端口被占用

#### 后端端口 (8000) 被占用

```powershell
# 查看端口占用
netstat -ano | Select-String ":8000"

# 关闭占用进程（替换 PID 为实际进程ID）
taskkill /F /PID <PID>
```

如无法关闭，可更换端口：

```powershell
python -m uvicorn app.main:app --host 0.0.0.0 --port 8001
```

#### 前端端口 (5173) 被占用

Vite 会自动尝试下一个可用端口 (5174, 5175...)，或手动指定端口：

```powershell
npm run dev -- --port 3000
```

## 认证流程

### 1. 注册用户

```powershell
Invoke-WebRequest -Uri "http://localhost:8000/api/auth/register" `
    -Method POST `
    -ContentType "application/json" `
    -Body '{"username":"testuser","password":"test123"}'
```

### 2. 登录获取 Token

```powershell
Invoke-WebRequest -Uri "http://localhost:8000/api/auth/login" `
    -Method POST `
    -ContentType "application/x-www-form-urlencoded" `
    -Body "username=testuser&password=test123"
```

返回示例：

```json
{"access_token":"eyJhbGci...","token_type":"bearer"}
```

### 3. 使用 Token 访问受保护接口

```powershell
Invoke-WebRequest -Uri "http://localhost:8000/api/auth/me" `
    -Method GET `
    -Headers @{"Authorization"="Bearer YOUR_TOKEN_HERE"}
```

## 代码修改记录

### 1. security.py - bcrypt 兼容处理

由于 bcrypt 5.0.0 与 passlib 不兼容，修改 `app/core/security.py`:

```python
# 原始代码 (有问题)
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def verify_password(plain_password: str, hashed_password: str) -> bool:
    return pwd_context.verify(plain_password, hashed_password)

def get_password_hash(password: str) -> str:
    return pwd_context.hash(password)

# 修改后
import bcrypt

def verify_password(plain_password: str, hashed_password: str) -> bool:
    password_bytes = plain_password[:72].encode('utf-8')
    hashed_bytes = hashed_password.encode('utf-8') if isinstance(hashed_password, str) else hashed_password
    return bcrypt.checkpw(password_bytes, hashed_bytes)

def get_password_hash(password: str) -> str:
    password_bytes = password[:72].encode('utf-8')
    return bcrypt.hashpw(password_bytes, bcrypt.gensalt()).decode('utf-8')
```

### 2. note.py - Pydantic V2 类型修复

修改 `app/schemas/note.py` 解决 Optional\[List] 默认值问题:

```python
# 原始代码 (有问题)
class NoteCreate(NoteBase):
    tag_ids: Optional[List[str]] = []

# 修改后
class NoteCreate(NoteBase):
    tag_ids: Optional[Any] = None
```

### 3. notes.py - 处理 tag\_ids 类型转换

在 `app/api/routes notes.py` 的 create\_note 函数中添加类型处理:

```python
tag_ids = note_data.tag_ids
if isinstance(tag_ids, str):
    import json as json_module
    try:
        tag_ids = json_module.loads(tag_ids)
    except:
        tag_ids = []
if not isinstance(tag_ids, list):
    tag_ids = []
```

### 4. note.py / user.py - 添加用户关联

为支持多用户笔记隔离，在 `app/models/note.py` 添加 user\_id 字段:

```python
class Note(Base, TimestampMixin):
    __tablename__ = "notes"

    id = Column(String(36), primary_key=True, default=generate_uuid)
    title = Column(String(500), nullable=False)
    content = Column(Text, nullable=True)
    folder_id = Column(String(36), ForeignKey("folders.id"), nullable=True)
    user_id = Column(String(36), ForeignKey("users.id"), nullable=True)
    linked_note_ids = Column(Text, nullable=True)

    folder = relationship("Folder", back_populates="notes")
    tags = relationship("Tag", secondary=note_tags, back_populates="notes")
    user = relationship("User", back_populates="notes")
```

在 `app/models/user.py` 添加反向关联:

```python
from sqlalchemy.orm import relationship

class User(Base, TimestampMixin):
    # ... existing fields ...
    notes = relationship("Note", back_populates="user")
```

### 5. 数据库迁移 - 添加 user\_id 列

如果遇到 `table notes has no column named user_id` 错误，需要手动添加列:

```python
import sqlite3
conn = sqlite3.connect('d:/知枢星图/backend/data/knowledge.db')
conn.execute('ALTER TABLE notes ADD COLUMN user_id VARCHAR(36)')
conn.commit()
conn.close()
```

### 6. notes.py - 修复 Pydantic 验证错误

由于数据库中 `linked_note_ids` 存储为 JSON 字符串，但 Pydantic 模型期望 list 类型，在所有返回笔记的接口中改用手动构建字典:

```python
# 避免使用 NoteResponse.model_validate()
note_dict = {
    "id": note.id,
    "title": note.title,
    "content": note.content,
    "folder_id": note.folder_id,
    "linked_note_ids": json.loads(note.linked_note_ids) if note.linked_note_ids else [],
    "tags": [],
    "created_at": note.created_at,
    "updated_at": note.updated_at
}
```

### 7. categories.py / skills.py - 添加 Response 导入

部分路由文件缺少 `Response` 类的导入，需要添加:

```python
from app.schemas.response import Response
```

## 故障排查

### 问题 1: start.bat 窗口闪退

**原因**：启动脚本报错导致窗口关闭，无法看到错误信息

**解决**：

1. 在终端中手动运行启动命令查看错误
2. 或右键编辑 start.bat，确保内容正确：

```batch
@echo off
chcp 65001 >nul
cd /d D:\知枢星图\backend
set PYTHONPATH=D:\知枢星图\backend
F:\ANACONDA\python.exe -m uvicorn app.main:app --reload --port 8000
```

### 问题 2: ImportError: No module named 'jose' 或 'uvicorn'

**原因**：依赖包未安装或安装到错误的 Python 环境

**解决**：以管理员身份运行 Anaconda Prompt，执行：

```bash
pip install python-jose cryptography passlib python-multipart bcrypt uvicorn fastapi
```

### 问题 3: bcrypt 错误

**原因**：bcrypt 5.0.0 与 passlib 不兼容

**解决**：参考上面"代码修改记录"部分

### 问题 4: Pydantic 类型验证错误

**原因**：Pydantic V2 对 Optional\[List] 默认值 \[] 的处理不同

**解决**：使用 Optional\[Any] 或在代码中动态处理类型

### 问题 5: 端口被占用

**解决**：更换端口

```powershell
python -m uvicorn app.main:app --host 0.0.0.0 --port 8001
```

### 问题 5: table notes has no column named user\_id

**原因**：数据库表结构与模型不匹配

**解决**：参考上面"代码修改记录"第5部分，添加数据库列

### 问题 6: Pydantic ValidationError: linked\_note\_ids

**原因**：数据库中 linked\_note\_ids 存储为 JSON 字符串，但 Pydantic 期望 list 类型

**解决**：参考上面"代码修改记录"第6部分，使用手动构建字典

### 问题 7: 前端无法连接后端 API

**原因**：后端未启动或跨域问题

**解决**：

1. 确保后端已启动并运行在 <http://localhost:8000>
2. 检查 vite.config.ts 中的代理配置是否正确
3. 查看浏览器控制台是否有 CORS 错误

### 问题 8: npm install 失败

**原因**：网络问题或 npm 源访问慢

**解决**：切换到国内镜像源

```powershell
# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 或使用 cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

### 问题 9: 前端页面空白

**原因**：可能是路由配置或编译错误

**解决**：

1. 检查浏览器控制台错误信息
2. 确保所有依赖已正确安装
3. 尝试清除缓存重新安装：`rm -rf node_modules && npm install`

### 问题 10: ModuleNotFoundError: No module named 'app'

**原因**：Python 无法找到 app 模块，PYTHONPATH 未正确设置

**解决**：在启动前设置 PYTHONPATH 环境变量

```powershell
cd d:\知枢星图\backend
$env:PYTHONPATH = "D:\知枢星图\backend"
F:\ANACONDA\python.exe -m uvicorn app.main:app --reload --port 8000
```

### 问题 11: NameError: name 'Response' is not defined

**原因**：部分路由文件缺少 `Response` 类的导入

**解决**：在相关文件中添加导入语句

```python
from app.schemas.response import Response
```

受影响的文件：

- `app/api/routes/categories.py`
- `app/api/routes/skills.py`

## 技术栈

### 后端

- **框架**: FastAPI
- **数据库**: SQLite (SQLAlchemy ORM)
- **认证**: JWT (python-jose)
- **密码加密**: bcrypt

### 前端

- **框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **状态管理**: Pinia
- **路由**: Vue Router
- **UI 组件库**: Naive UI
- **样式**: Tailwind CSS + SCSS
- **HTTP 客户端**: Axios
- **Markdown 渲染**: markdown-it
