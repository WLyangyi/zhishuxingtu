## 1. 系统概述

### 1.1 产品定位

知枢星图是一个现代化的**个人知识资产管理平台**，帮助用户整合知识资源、建立知识关联、通过 AI 技术实现智能问答和知识检索。

## 4. 核心模块设计

### 4.1 认证模块

**架构位置**: `backend/app/api/routes/auth.py`

**功能流程**:
```
用户登录请求
    │
    ▼
验证用户名密码 (bcrypt)
    │
    ▼
生成 JWT Token (HS256, 7天有效期)
    │
    ▼
返回 Token 给客户端
```

**安全特性**:
- 密码使用 bcrypt 加密
- JWT Token 认证
- 支持跨域配置

### 4.2 笔记管理模块

**架构位置**: `backend/app/api/routes/notes.py`

**核心功能**:
| 功能 | API 端点 | 说明 |
|------|----------|------|
| 创建笔记 | POST /api/notes | 支持 Markdown、内容、文件夹、标签 |
| 获取笔记列表 | GET /api/notes | 支持分页、过滤、排序 |
| 获取笔记详情 | GET /api/notes/:id | 包含反向链接 |
| 更新笔记 | PUT /api/notes/:id | 自动更新关联的链接 |
| 删除笔记 | DELETE /api/notes/:id | 级联删除关联 |
| 获取反向链接 | GET /api/notes/:id/backlinks | 查询引用当前笔记的所有笔记 |

### 4.3 双向链接模块

**实现机制**:
1. **链接解析**: 使用正则表达式 `\[\[([^\]]+)\]\]` 解析笔记内容中的双向链接
2. **链接存储**: 将解析出的链接目标笔记 ID 存储在 `linked_note_ids` 字段（JSON 格式）
3. **反向链接查询**: 根据目标笔记 ID 查询所有源笔记

**链接数据结构**:
```json
{
  "linked_note_ids": ["note-uuid-1", "note-uuid-2"]
}
```

### 4.4 知识图谱模块

**架构位置**:
- 前端: `frontend/src/views/GraphView.vue`
- 后端: `backend/app/api/routes/graph.py`

**图谱类型**:
| 类型 | 端点 | 说明 |
|------|------|------|
| 全局图谱 | GET /api/graph/global | 显示所有笔记的关系网络 |
| 局部图谱 | GET /api/graph/local/:id | 显示单个笔记的直接关联 |

**节点数据结构**:
```typescript
interface GraphNode {
  id: string;
  label: string;
  type: 'note' | 'folder' | 'tag';
  color?: string;
}

interface GraphLink {
  source: string;
  target: string;
}
```

**可视化实现**: 使用 D3.js 力导向图实现交互式图谱

### 4.5 AI 问答模块 (RAG)

**架构位置**:
- 服务层: `backend/app/services/vector_store.py`
- 嵌入服务: `backend/app/services/embedding_service.py`
- API: `backend/app/api/routes/search.py`

**RAG 流程**:
```
用户问题
    │
    ▼
文本向量化 (通义千问 text-embedding-v3, 1024维)
    │
    ▼
FAISS 向量检索 (Top-K 相关笔记)
    │
    ▼
构建提示词 (问题 + 相关笔记内容)
    │
    ▼
调用 LLM API (qwen-turbo / GPT-4 / Claude)
    │
    ▼
返回 AI 答案 + 相关笔记引用
```

**向量配置**:
| 配置项 | 值 |
|--------|-----|
| 模型 | 通义千问 text-embedding-v3 |
| 向量维度 | 1024 |
| 检索阈值 | 0.3 |
| Top-K | 5 |
| API 来源 | 阿里云 DashScope |

### 4.6 分类体系模块 (V2)

**架构位置**: `backend/app/api/routes/categories.py`

**分类层次**:
```
分类 (Category)
    │
    └── 内容类型 (ContentType)
            │
            └── 内容 (Content)
```

**预设分类**:
| 分类 | 内容类型 | 说明 |
|------|----------|------|
| 个人 (Personal) | 笔记、日记、简历 | 个人成长相关内容 |
| 工作 (Work) | 业务知识、管理知识 | 职业发展内容 |
| 素材 (Assets) | 图片、链接、视频 | 创作资源 |

### 4.7 Skill 智能模块 (V3)

**架构位置**: `backend/app/api/routes/skills.py`

**Skill 执行流程**:
```
用户触发 / 定时调度
    │
    ▼
读取 Skill 定义 (输入参数、执行逻辑)
    │
    ▼
从知识库获取相关内容
    │
    ▼
调用 AI 执行 Skill
    │
    ▼
将结果存回知识库 (可选)
    │
    ▼
记录执行历史
```

### 4.8 提示词管理模块 (V3.1) ✅ 新增

**架构位置**:
- 后端: `backend/app/api/routes/prompts.py`
- 前端: `frontend/src/views/PromptsView.vue`

**核心功能**:
| 功能 | 说明 |
|------|------|
| 提示词模板管理 | CRUD 操作，支持系统预设和用户自定义 |
| 变量替换 | 支持 `{variable_name}` 格式的动态变量 |
| 敏感词过滤 | 自动检测并过滤不当内容 |
| 免责声明 | AI 回复自动追加免责声明 |

**预置提示词模板**:
| 模板名称 | 分类 | 主要变量 |
|----------|------|----------|
| 知识问答 | ai_chat | current_date, current_weekday, context, question |
| AI搜索问答 | ai_search | current_date, current_weekday, context, question |
| 对话总结 | skill | conversation |
| 每日资讯 | skill | sources, keywords |
| 简历解析 | skill | resume_text |

**数据结构**:

{
  "id": "uuid",

services/

    ├── encode()         # 文本向量化
    ├── encode_batch()   # 批量向量化

```

OPENAI_BASE_URL      # API 地址

DASHSCOPE_EMBEDDING_MODEL  # 嵌入模型 (text-embedding-v3)
FAISS_INDEX_PATH      # FAISS 索引路径
```
