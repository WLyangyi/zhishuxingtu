## 1. 系统概述

### 1.1 产品定位

知枢星图是一个现代化的**个人知识资产管理平台**，帮助用户整合知识资源、建立知识关联、通过 AI 技术实现智能问答和知识检索。

## 4. 核心模块设计

### 4.1 认证模块

**架构位置**: `backend/app/api/routes/auth.py`

**功能流程**:

```
用户登录请求
    │
    ▼
验证用户名密码 (bcrypt)
    │
    ▼
生成 JWT Token (HS256, 7天有效期)
    │
    ▼
返回 Token 给客户端
```

**安全特性**:

- 密码使用 bcrypt 加密
- JWT Token 认证
- 支持跨域配置

### 4.2 笔记管理模块

**架构位置**: `backend/app/api/routes/notes.py`

**核心功能**:

| 功能     | API 端点                       | 说明                    |
| ------ | ---------------------------- | --------------------- |
| 创建笔记   | POST /api/notes              | 支持 Markdown、内容、文件夹、标签 |
| 获取笔记列表 | GET /api/notes               | 支持分页、过滤、排序            |
| 获取笔记详情 | GET /api/notes/:id           | 包含反向链接                |
| 更新笔记   | PUT /api/notes/:id           | 自动更新关联的链接             |
| 删除笔记   | DELETE /api/notes/:id        | 级联删除关联                |
| 获取反向链接 | GET /api/notes/:id/backlinks | 查询引用当前笔记的所有笔记         |

### 4.3 双向链接模块

**实现机制**:

1. **链接解析**: 使用正则表达式 `\[\[([^\]]+)\]\]` 解析笔记内容中的双向链接
2. **链接存储**: 将解析出的链接目标笔记 ID 存储在 `linked_note_ids` 字段（JSON 格式）
3. **反向链接查询**: 根据目标笔记 ID 查询所有源笔记

**链接数据结构**:

```json
{
  "linked_note_ids": ["note-uuid-1", "note-uuid-2"]
}
```

### 4.4 知识图谱模块

**架构位置**:

- 前端: `frontend/src/views/GraphView.vue`
- 后端: `backend/app/api/routes/graph.py`

**图谱类型**:

| 类型   | 端点                       | 说明          |
| ---- | ------------------------ | ----------- |
| 全局图谱 | GET /api/graph/global    | 显示所有笔记的关系网络 |
| 局部图谱 | GET /api/graph/local/:id | 显示单个笔记的直接关联 |

**节点数据结构**:

```typescript
interface GraphNode {
  id: string;
  label: string;
  type: 'note' | 'folder' | 'tag';
  color?: string;
}

interface GraphLink {
  source: string;
  target: string;
}
```

**可视化实现**: 使用 D3.js 力导向图实现交互式图谱

### 4.5 AI 问答模块 (RAG) - 当前实现

**架构位置**:

- 服务层: `backend/app/services/vector_store.py`
- 嵌入服务: `backend/app/services/embedding_service.py`
- API: `backend/app/api/routes/search.py`

**RAG 流程（当前手工实现）**:

```
用户问题
    │
    ▼
文本向量化 (通义千问 text-embedding-v3, 1024维)
    │
    ▼
FAISS 向量检索 (Top-K 相关笔记)
    │
    ▼
构建提示词 (问题 + 相关笔记内容)
    │
    ▼
调用 LLM API (qwen-turbo / GPT-4 / Claude)
    │
    ▼
返回 AI 答案 + 相关笔记引用
```

**向量配置**:

| 配置项    | 值                      |
| ------ | ---------------------- |
| 模型     | 通义千问 text-embedding-v3 |
| 向量维度   | 1024                   |
| 检索阈值   | 0.3                    |
| Top-K  | 5                      |
| API 来源 | 阿里云 DashScope          |

### 4.5.1 AI 问答模块 (RAG) - LangChain 实现 ✅ 已完成

> **已完成**：LangChain 框架集成完成，支持渐进式迁移。

**LangChain RAG 流程**:

```
用户问题
    │
    ▼
LangChain Embeddings (通义千问 text-embedding-v3)
    │
    ▼
LangChain VectorStore (FAISS) 检索
    │
    ▼
LangChain RAGChain / ChatChain
    │
    ├── ChatPromptTemplate (模板化管理)
    ├── LLM 调用 (qwen-turbo)
    └── 对话历史管理 (可选)
    │
    ▼
返回 AI 答案 + 相关笔记引用
```

**LangChain 组件映射**:

| LangChain 组件           | 对应功能     | 状态    |
| ---------------------- | -------- | ----- |
| `LangChainEmbeddings`  | 文本向量化    | ✅ 已实现 |

| `RAGChain`             | RAG 问答   | ✅ 已实现 |
| `ChatChain`            | 多轮对话     | ✅ 已实现 |

**技术原理**:

```
传统模式（同步）:
用户问题 → 后端处理 → 等待完整响应 → 一次性返回

流式模式（SSE）:
用户问题 → 后端处理 → 逐 token 推送 → 前端实时渲染
                    ↓
              data: {"content": "你"}
              data: {"content": "好"}
              data: {"content": "！"}
              ...
```

**实现流程**:

```
前端发起请求 (Accept: text/event-stream)
    │
    ▼
后端创建 SSE 连接 (StreamingResponse)
    │
    ▼
调用 LLM API (stream=True)
    │
    ▼
逐 chunk 解析并推送
    │
    ├── data: {"type": "content", "text": "..."}
    ├── data: {"type": "content", "text": "..."}
    ├── data: {"type": "sources", "notes": [...]}
    └── data: [DONE]
    │
    ▼
前端 EventSource 接收并渲染
```


```python
from rank_bm25 import BM25Okapi
import jieba

class BM25Service:
    def __init__(self):
        self.tokenized_corpus = []
        self.bm25 = None
        self.doc_ids = []
    
    def index_documents(self, docs: List[Dict]):
        """构建 BM25 索引"""
        self.doc_ids = [doc['id'] for doc in docs]
        self.tokenized_corpus = [jieba.lcut(doc['content']) for doc in docs]
        self.bm25 = BM25Okapi(self.tokenized_corpus)
    
    def search(self, query: str, top_k: int = 15) -> List[Tuple[str, float]]:
        """BM25 检索"""
        query_tokens = jieba.lcut(query)
        scores = self.bm25.get_scores(query_tokens)
        
        doc_scores = list(zip(self.doc_ids, scores))
        doc_scores.sort(key=lambda x: x[1], reverse=True)
        
        return doc_scores[:top_k]
```

**RRF 融合实现**：

```python
def reciprocal_rank_fusion(
    vector_results: List[Tuple[str, float]],
    bm25_results: List[Tuple[str, float]],
    k: int = 60,
    vector_weight: float = 0.6,
    bm25_weight: float = 0.4
) -> List[Tuple[str, float]]:
    """RRF 混合融合"""
    rrf_scores = {}
    
    # 向量检索结果
    for rank, (doc_id, score) in enumerate(vector_results):
        rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + vector_weight * (1 / (k + rank + 1))
    
    # BM25 结果
    for rank, (doc_id, score) in enumerate(bm25_results):
        rrf_scores[doc_id] = rrf_scores.get(doc_id, 0) + bm25_weight * (1 / (k + rank + 1))
    
    # 排序
    sorted_results = sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)
    return sorted_results
```

**CLIP 服务实现**：

```python
from transformers import CLIPProcessor, CLIPModel
from PIL import Image
import torch

class CLIPService:
    def __init__(self, model_name: str = "openai/clip-vit-base-patch32"):
        self.model = CLIPModel.from_pretrained(model_name)
        self.processor = CLIPProcessor.from_pretrained(model_name)
        self.device = "cuda" if torch.cuda.is_available() else "cpu"
        self.model.to(self.device)
    
    def encode_image(self, image_path: str) -> np.ndarray:
        """图片向量化"""
        image = Image.open(image_path).convert("RGB")
        inputs = self.processor(images=image, return_tensors="pt").to(self.device)
        
        with torch.no_grad():
            image_features = self.model.get_image_features(**inputs)
        
        return image_features.cpu().numpy()[0]
    
    def encode_text(self, text: str) -> np.ndarray:
        """文本向量化"""
        inputs = self.processor(text=[text], return_tensors="pt", padding=True).to(self.device)
        
        with torch.no_grad():
            text_features = self.model.get_text_features(**inputs)
        
        return text_features.cpu().numpy()[0]
    
    def similarity(self, image_path: str, text: str) -> float:
        """计算图片-文本相似度"""
        image_features = self.encode_image(image_path)
        text_features = self.encode_text(text)
        
        # 归一化
        image_features = image_features / np.linalg.norm(image_features)
        text_features = text_features / np.linalg.norm(text_features)
        
        # 余弦相似度
        return np.dot(image_features, text_features)
```

```python
        scores, indices = self.faiss_index.search(query_vector, top_k)
        
        results = []
        for score, idx in zip(scores[0], indices[0]):
            if idx < len(self.image_ids):
                img_id = self.image_ids[idx]
                results.append({
                    "image_id": img_id,
                    "score": float(score),
                    "metadata": self.image_metadata[img_id]
                })
        
        return results
```

**图片问答实现**：

```python
class MultimodalQA:
    def __init__(self):
        self.clip_service = CLIPService()
        self.llm_service = get_llm_service()
    
    async def answer_about_image(self, question: str, image_path: str) -> str:
        """基于图片内容回答问题"""
        # 1. 分析图片内容
        image_description = self._describe_image(image_path)
        
        # 2. 构建提示词
        prompt = f"""根据图片内容回答问题。

图片描述：{image_description}

问题：{question}

请基于图片内容回答问题。"""
        
        # 3. 调用 LLM
        response = await self.llm_service.invoke([
            {"role": "user", "content": prompt}
        ])
        
        return response.content
    
    def _describe_image(self, image_path: str) -> str:
        """使用 CLIP 分析图片主要物体"""
        # CLIP 零样本分类
        candidates = [
            "a photo containing a cat",
            "a photo containing a dog", 
            "a photo containing a landscape",
            "a photo containing a person",
            "a photo containing text or code",
            "a photo containing food",
            "a photo containing a building"
        ]
        
        image = Image.open(image_path).convert("RGB")
        inputs = self.clip_service.processor(text=candidates, images=image, return_tensors="pt", padding=True)
        
        with torch.no_grad():
            outputs = self.clip_service.model(**inputs)
        
        logits_per_image = outputs.logits_per_image
        probs = logits_per_image.softmax(dim=1)
        
        top_idx = probs.argmax().item()
        return f"这张图片最可能是{candidates[top_idx].replace('a photo containing a ', '')}"
```
