## 1. 系统概述

### 1.1 产品定位

知枢星图是一个现代化的**个人知识资产管理平台**，帮助用户整合知识资源、建立知识关联、通过 AI 技术实现智能问答和知识检索。

## 4. 核心模块设计

### 4.1 认证模块

**架构位置**: `backend/app/api/routes/auth.py`

**功能流程**:
```
用户登录请求
    │
    ▼
验证用户名密码 (bcrypt)
    │
    ▼
生成 JWT Token (HS256, 7天有效期)
    │
    ▼
返回 Token 给客户端
```

**安全特性**:
- 密码使用 bcrypt 加密
- JWT Token 认证
- 支持跨域配置

### 4.2 笔记管理模块

**架构位置**: `backend/app/api/routes/notes.py`

**核心功能**:
| 功能 | API 端点 | 说明 |
|------|----------|------|
| 创建笔记 | POST /api/notes | 支持 Markdown、内容、文件夹、标签 |
| 获取笔记列表 | GET /api/notes | 支持分页、过滤、排序 |
| 获取笔记详情 | GET /api/notes/:id | 包含反向链接 |
| 更新笔记 | PUT /api/notes/:id | 自动更新关联的链接 |
| 删除笔记 | DELETE /api/notes/:id | 级联删除关联 |
| 获取反向链接 | GET /api/notes/:id/backlinks | 查询引用当前笔记的所有笔记 |

### 4.3 双向链接模块

**实现机制**:
1. **链接解析**: 使用正则表达式 `\[\[([^\]]+)\]\]` 解析笔记内容中的双向链接
2. **链接存储**: 将解析出的链接目标笔记 ID 存储在 `linked_note_ids` 字段（JSON 格式）
3. **反向链接查询**: 根据目标笔记 ID 查询所有源笔记

**链接数据结构**:
```json
{
  "linked_note_ids": ["note-uuid-1", "note-uuid-2"]
}
```

### 4.4 知识图谱模块

**架构位置**:
- 前端: `frontend/src/views/GraphView.vue`
- 后端: `backend/app/api/routes/graph.py`

**图谱类型**:
| 类型 | 端点 | 说明 |
|------|------|------|
| 全局图谱 | GET /api/graph/global | 显示所有笔记的关系网络 |
| 局部图谱 | GET /api/graph/local/:id | 显示单个笔记的直接关联 |

**节点数据结构**:
```typescript
interface GraphNode {
  id: string;
  label: string;
  type: 'note' | 'folder' | 'tag';
  color?: string;
}

interface GraphLink {
  source: string;
  target: string;
}
```

**可视化实现**: 使用 D3.js 力导向图实现交互式图谱

### 4.5 AI 问答模块 (RAG) - 当前实现

**架构位置**:
- 服务层: `backend/app/services/vector_store.py`
- 嵌入服务: `backend/app/services/embedding_service.py`
- API: `backend/app/api/routes/search.py`

**RAG 流程（当前手工实现）**:
```
用户问题
    │
    ▼
文本向量化 (通义千问 text-embedding-v3, 1024维)
    │
    ▼
FAISS 向量检索 (Top-K 相关笔记)
    │
    ▼
构建提示词 (问题 + 相关笔记内容)
    │
    ▼
调用 LLM API (qwen-turbo / GPT-4 / Claude)
    │
    ▼
返回 AI 答案 + 相关笔记引用
```

**向量配置**:
| 配置项 | 值 |
|--------|-----|
| 模型 | 通义千问 text-embedding-v3 |
| 向量维度 | 1024 |
| 检索阈值 | 0.3 |
| Top-K | 5 |
| API 来源 | 阿里云 DashScope |

### 4.5.1 AI 问答模块 (RAG) - LangChain 实现 ✅ 已完成

> **已完成**：LangChain 框架集成完成，支持渐进式迁移。

**LangChain RAG 流程**:
```
用户问题
    │
    ▼
LangChain Embeddings (通义千问 text-embedding-v3)
    │
    ▼
LangChain VectorStore (FAISS) 检索
    │
    ▼
LangChain RAGChain / ChatChain
    │
    ├── ChatPromptTemplate (模板化管理)
    ├── LLM 调用 (qwen-turbo)
    └── 对话历史管理 (可选)
    │
    ▼
返回 AI 答案 + 相关笔记引用
```

**技术原理**:
```
传统模式（同步）:
用户问题 → 后端处理 → 等待完整响应 → 一次性返回

流式模式（SSE）:
用户问题 → 后端处理 → 逐 token 推送 → 前端实时渲染
                    ↓
              data: {"content": "你"}
              data: {"content": "好"}
              data: {"content": "！"}
              ...
```

**实现流程**:
```
前端发起请求 (Accept: text/event-stream)
    │
    ▼
后端创建 SSE 连接 (StreamingResponse)
    │
    ▼
调用 LLM API (stream=True)
    │
    ▼
逐 chunk 解析并推送
    │
    ├── data: {"type": "content", "text": "..."}
    ├── data: {"type": "content", "text": "..."}
    ├── data: {"type": "sources", "notes": [...]}
    └── data: [DONE]
    │
    ▼
前端 EventSource 接收并渲染
```

**配置参数**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `USE_STREAMING` | true | 是否启用流式输出 |
| `STREAM_CHUNK_SIZE` | 20 | 每次推送的字符数 |
| `STREAM_TIMEOUT` | 60 | 流式连接超时时间（秒） |

**API 端点**:
```

POST /api/search/chat/stream  # 流式多轮对话
```

  async connect(url: string, body: any, onMessage: (msg: SSEMessage) => void) {
    const response = await fetch(url, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(body)
    });
    
    const reader = response.body.getReader();
    const decoder = new TextDecoder();

      if (done) break;
