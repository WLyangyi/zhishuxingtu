### 2.3 数据库模型实现（已完成）

#### users 表
```sql
CREATE TABLE users (
    id VARCHAR(36) PRIMARY KEY,
    username VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### folders 表
```sql
CREATE TABLE folders (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    parent_id VARCHAR(36),
    level INTEGER CHECK (level IN (0, 1, 2, 3)),  -- 扩展至4层
    category_id VARCHAR(36),                       -- 新增：所属分类
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES folders(id),
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
```

#### notes 表
```sql
CREATE TABLE notes (
    id VARCHAR(36) PRIMARY KEY,
    title VARCHAR(500) NOT NULL,
    content TEXT,
    folder_id VARCHAR(36),
    user_id VARCHAR(36),
    linked_note_ids TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (folder_id) REFERENCES folders(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### tags 表
```sql
CREATE TABLE tags (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL UNIQUE,
    color VARCHAR(7) DEFAULT '#00d4ff',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### note_tags 表
```sql
CREATE TABLE note_tags (
    note_id VARCHAR(36),
    tag_id VARCHAR(36),
    PRIMARY KEY (note_id, tag_id),
    FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
    FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);
```

- 使用正则表达式 `\[\[([^\]]+)\]\]` 解析链接

- 使用 vue-force-graph 库渲染

- 主页简化为 3 个分类入口（个人/工作/素材）
- 文件夹层级从 2 层扩展至 4 层
- 各分类支持不同的内容类型和存储方式
- AI 助手作为「个人」入口内的功能模块

### 8.6 文件夹层级扩展

**原有设计：** 最多 2 层（Level 0 顶层 + Level 1 子文件夹）

**新设计：** 最多 4 层（Level 0 → Level 1 → Level 2 → Level 3）

```
个人
├── 笔记文件夹 A（Level 0）
│   ├── 子文件夹 A1（Level 1）
│   │   ├── 子子文件夹 A1a（Level 2）
│   │   │   └── 深层文件夹 A1a1（Level 3）
│   │   └── 子子文件夹 A1b（Level 2）
│   └── 子文件夹 A2（Level 1）
└── 笔记文件夹 B（Level 0）
```

## 9. 后端数据模型更新

### 9.1 folders 表更新

**变更内容：** `level` 字段从 `CHECK (level IN (0, 1))` 改为 `CHECK (level IN (0, 1, 2, 3))`

```sql
CREATE TABLE folders (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    parent_id VARCHAR(36),
    level INTEGER CHECK (level IN (0, 1, 2, 3)),  -- 扩展至4层
    category_id VARCHAR(36),                       -- 新增：所属分类
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES folders(id),
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
```

### 9.2 categories 表（内容分类）

```sql
CREATE TABLE categories (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    icon VARCHAR(50),
    color VARCHAR(7) DEFAULT '#00d4ff',
    sort_order INTEGER DEFAULT 0,
    is_system BOOLEAN DEFAULT FALSE,  -- 系统预设分类不可删除
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

**预置数据：**
| 分类ID | 名称 | 图标 | 系统预设 |
|--------|------|------|----------|
| personal | 个人 | 👤 | 是 |
| work | 工作 | 💼 | 是 |
| assets | 素材 | 📦 | 是 |

### 9.3 content_types 表（内容类型）

```sql
CREATE TABLE content_types (
    id VARCHAR(36) PRIMARY KEY,
    category_id VARCHAR(36) NOT NULL,
    name VARCHAR(100) NOT NULL,
    icon VARCHAR(50),
    color VARCHAR(7) DEFAULT '#00d4ff',
    field_schema TEXT,              -- JSON Schema，定义扩展字段
    is_system BOOLEAN DEFAULT FALSE,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES categories(id)
);
```

**预置数据：**
| 分类 | 内容类型 | 图标 |
|------|---------|------|
| 个人 | 笔记 | 📝 |
| 个人 | 日记 | 📅 |
| 个人 | 简历 | 📄 |
| 工作 | 项目文档 | 📋 |
| 工作 | 任务列表 | ✅ |
| 工作 | 里程碑 | 📊 |
| 素材 | 图片 | 🖼️ |
| 素材 | 附件 | 📎 |
| 素材 | 链接收藏 | 🔗 |

### 9.4 contents 表（统一内容表）

```sql
CREATE TABLE contents (
    id VARCHAR(36) PRIMARY KEY,
    type_id VARCHAR(36) NOT NULL,          -- 内容类型ID
    category_id VARCHAR(36) NOT NULL,      -- 所属分类ID
    title VARCHAR(500) NOT NULL,
    content TEXT,
    extra_fields TEXT,                     -- JSON，存储扩展字段
    linked_content_ids TEXT,              -- JSON，关联的内容ID列表
    file_path VARCHAR(500),                -- 本地文件路径
    file_url VARCHAR(1000),               -- 外部链接
    file_size INTEGER,                     -- 文件大小
    folder_id VARCHAR(36),                 -- 所属文件夹
    user_id VARCHAR(36),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (type_id) REFERENCES content_types(id),
    FOREIGN KEY (category_id) REFERENCES categories(id),
    FOREIGN KEY (folder_id) REFERENCES folders(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```
